Three jobs. One runtime.

Baton doesn't replace your models or your tools. It's the layer that decides what runs, in what order, and what happens when something fails.

Plan

Break down the work

Describe an outcome in plain language. Baton turns it into a workflow graph — which model handles what, what runs in parallel, what waits on what.

Route

Send each step to the right place

Every step goes to the model or tool best suited for it, with automatic fallback if a call fails, times out, or hits a rate limit.

Execute

Run it, watch it, trust it

Workflows run on a schedule, a trigger, or on demand. Every step is logged, so when something breaks, you see exactly where and why.

How it works

From idea to running workflow

01

Connect your tools

Link the models, databases, and apps you already use. Baton reads their capabilities so it knows what each one is good for.

02

Describe the workflow

Write what you want to happen, step by step or as a single goal. Baton drafts the plan and shows you the graph before anything runs.

03

Review the routing

See which model or agent handles each step, adjust any assignment, and set fallback rules for failures or rate limits.

04

Run and monitor

Trigger it manually, on a schedule, or from an event. Every run is logged step by step, so debugging takes minutes, not hours.
